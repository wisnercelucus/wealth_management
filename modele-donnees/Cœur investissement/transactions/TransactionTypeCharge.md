# TransactionTypeCharge

Which charge names a transaction type **declares** — this is what makes manual charge
fields appear on the entry form when the type is selected. Declared fields are the
**deliberate** side of charging: the operator enters a value when there is a negotiated
or exceptional charge to take. Tariffs ([ChargePolicy](../charges/ChargePolicy.md))
need no declaration and never touch these fields — the two sources are strictly
separate and additive. The charge vocabulary lives in the
[Charges module](../charges/README.md); this table is part of the **type's**
declaration surface, beside [contract lines](TransactionTypeLeg.md) and
[eligibility](TransactionTypeEligibility.md).

## Essential fields

| Field | Type | Req | Description |
|-------|------|-----|-------------|
| `id` | uuid | ● | Surrogate primary key. |
| `transactionTypeId` | FK→TransactionType | ● | The owning type. |
| `chargeNameId` | FK→[ChargeName](../charges/ChargeName.md) | ● | The declared charge. Never a tax-target name — taxes are derived from the parent's binding, never entered. |
| `sequence` | int | ● | Display order on the form. |
| + envelope | | ● | See [README](README.md). |

`UNIQUE (transactionTypeId, chargeNameId)`.

## Notes & rules

- **On type creation the default set is seeded** (`COMMISSION`, `FEE`); the admin
  removes or adds names per type on the authoring screen.
- **Editing affects future entries only.** A declared charge is not a contract line —
  charge legs are auto-generated by the engine — so declaring or removing one on a
  type that has already posted **never requires a new type code**. Only changes to the
  type's own contract lines or `declaredAmounts` freeze
  ([TransactionType](TransactionType.md) immutability).
- The tax preview (e.g. TCA beside a `FEE` field) is derived read-only UI from the
  name's `taxChargeNameId` — never a second field.
- An empty field at entry = no legs = no charge, no tax: **absence is the waiver**
  (a permission-gated "clear" on pre-filled fields is optional UI, not a status).
  After posting, reversal only.

## Clean model

```
TransactionTypeCharge
  id                 uuid   PK
  transactionTypeId  FK TransactionType
  chargeNameId       FK ChargeName        -- never a tax-target name
  sequence           int
  + envelope (isActive, externalId, externalRef, createdAt, createdBy, modifiedAt, modifiedBy)
  unique (transactionTypeId, chargeNameId)
```
